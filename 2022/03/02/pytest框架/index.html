Pytest单元 / 自动化测试框架

一 、简要介绍

（1）什么是单元测试框架

单元测试是指在软件开发当中，针对软件的最小单位（函数方法）进行正确性的检查测试

（2）单元测试框架

java : junit / testing

python: unittest / pytest

（3）单元测试框架主要做什么

1.测试发现：从多个文件中找到测试用例

2.测试执行：按照一定的顺序和规则去执行，并生成结果

3.测试判断：通过断言判断预期结果和实际结果的差异

4.测试报告：统计测试进度，耗时。通过率，生成测试报告

二、单元测试框架和自动化测试框架的关系

1.提高测试效率，降低维护成本

2.减少人工干预，提高测试的准确性，增加代码的复用性

3.核心思想是让不懂代码的人能通过框架去实现自动化

pom设计模式 、数据驱动 、 关键字驱动、全局配置文件的封装 、 日志监控 、selenium ,requests二次封装 、断言 、报告邮件 …..

三、pytest简介

pytest是一个非常成熟的单元测试框架，比unittest更灵活，容易上手，pytest可以和selenium，requests,appium 结合实现web自动化，APP自动化，接口自动化，可以实现测试用例的跳过以及失败后reruns，可以和Jenkins持续集成

插件介绍：

pytest-html 生成html格式的自动化测试报告

pytest-xdist 测试用例分布式执行，多cpu分发

pytest-ordering 用于改变测试用例的执行顺序

pytest-rerunfailures 用例失败后重跑

allure-pytest 生成美观的测试报告

通过文本一键安装插件 pip install -r xx.txt
